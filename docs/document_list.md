## 4. ドキュメント管理の原則

1. **バージョン管理**: 全ドキュメントはGitHubでバージョン管理
2. **最新版の明示**: 本ドキュメントリストで最新版を一元管理
3. **階層関係の維持**: プロジェクト計画書 → WBS → SOP の階層を維持
4. **定期更新**: プロジェクト進行に伴い定期的に更新
5. **アーカイブ管理**: 重複・旧バージョンは明示的にアーカイブ
